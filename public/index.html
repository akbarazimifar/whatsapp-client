Deployment

You can choose any host that provides support for Node apps. We'll be deploying the bot to Heroku here.

Follow the steps below:

  1. Install Heroku CLI & make sure to do heroku login to connect your account.

  2. Create the heroku app. Choose the name you want while it's still available.

  3. Go to Dashboard | Heroku → <app-name> → Settings.
    Scroll down to Buildpacks and add the following buildpacks:

    https://github.com/jontewks/puppeteer-heroku-buildpack

  4. Navigate to Deploy → Deployment method. There are 3 options, choose the Github then authenticate with your repository.

  5. On the App connected to GitHub select the repository to deploy. Scroll down until you find the section Manual deploy, then click Deploy Branch.

Authentication

The app is now successfully deployed. By default, the web application is running, but for whatsapp-client it cannot run because it requires authentication with qrcode. So I'm so sure that this method will be safe because whatsapp-client will run when you ask to run.

Open your terminal and type:

heroku run bash -a <name-app>

If connected, run this command to make sure that the file contents are correct. Using lowercase L 

ls

Run app to get the QR Code to scan:

npm run start

Scan this QR code with WhatsApp Web on your phone.
If you get the message Client is ready!, you're all set to go!

To stop it, just press CTRL + C

To run it again:

npm run start

Session storage

whatsapp-client will automatically save session information into a file.

If you want to be more secure, I suggest that after you finish running whatsapp-client, you unlink the device in the WhatsApp section.
